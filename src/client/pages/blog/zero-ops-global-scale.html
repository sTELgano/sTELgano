Zero Ops at the Edge.

What it means to deploy globally without managing a single server, and how Cloudflare Workers turns “wrangler deploy” into a 330-city rollout.

2026-04-18 7 min read

What Zero Ops Actually Means

“Zero ops” does not mean zero work. It means the work you do is on the product, not on the infrastructure that runs it. Let’s be specific about what we used to own that we no longer do.

v1 — What We Managed
  • •OS security patches
  • •Erlang/OTP version upgrades
  • •PostgreSQL vacuuming and index bloat
  • •Let’s Encrypt renewal
  • •nginx configuration and logs
  • •Disk capacity monitoring
  • •Capacity planning for burst load
  • •Backup strategy and recovery drills
v2 — What We Manage
  • •The TypeScript source code
  • •D1 schema migrations
  • •wrangler.toml configuration
  • —Everything else: Cloudflare

wrangler deploy Is the Entire Pipeline

Deploying v1 to production involved: building an Elixir release in CI, scp-ing the tarball to the droplet, extracting it, running Stelgano.Release.migrate(), and bouncing the systemd unit. Multiple steps, multiple failure modes, a rollback procedure that required keeping the previous release tarball around.

Deploying v2:

$ wrangler deploy

That’s the whole pipeline. Wrangler bundles the Worker, uploads it to Cloudflare’s asset store, and propagates it to every PoP. Within seconds, the new code is live globally. Rollback is wrangler rollback.

Database migrations run separately with wrangler d1 migrations apply stelgano --remote. That’s the only manual step, and it’s idempotent.

Global by Default

v1 ran in a single DigitalOcean region. A user in Nairobi connecting to a server in Frankfurt experienced 150ms+ of baseline latency before any application logic ran. WebSocket connections amplify this: every message round-trip is at least two of those latencies.

Workers runs at Cloudflare’s global network: 330+ cities, spanning every continent. A user in Nairobi connects to Cloudflare’s Nairobi PoP. A user in São Paulo connects to a PoP in São Paulo. The Worker that handles the initial request runs close to the user by default.

Durable Objects are an important nuance here: because a DO must be globally unique, it lives in one specific Cloudflare datacenter. The first request to a room’s DO determines where it lives. Subsequent requests from geographically distant users are routed to that datacenter. For a two-person privacy channel, both parties are typically in the same region, so the DO naturally colocates with them.

Cold Starts: The V8 Advantage

Cold starts are the common objection to serverless compute. In container-based FaaS, a cold start means spinning up a container, loading a runtime, and initialising an application—hundreds of milliseconds to seconds. That’s a real problem.

Workers uses V8 isolates, not containers. An isolate is a lightweight JavaScript execution context—no OS process, no container image, no network stack bootstrapping. Cold start on a new isolate is measured in single-digit milliseconds, not hundreds. For our WebSocket upgrade path, this is inconsequential.

Durable Objects warm further because they hibernate rather than terminate. A room that hasn’t had activity for an hour will wake from hibernation—faster than a cold start because the DO’s durable state is preserved and doesn’t need to be re-initialised from scratch.

Observability Without a Monitoring Stack

Running your own server means running your own observability. Prometheus, Grafana, alerting rules, on-call rotation for a small team. Or you pay for a SaaS monitoring platform and add another credential rotation to the ops burden.

On Workers, the Cloudflare dashboard gives you request rates, error rates, CPU time, and latency distributions out of the box. Our custom metrics—rooms created, messages sent, country breakdown—live in Analytics Engine and are queryable via GraphQL from the admin dashboard. No Prometheus, no Grafana, no alerting configuration.

This isn’t a compromise. For a product at our scale, the built-in visibility is sufficient. We added instrumentation where the platform didn’t already provide it; everything else comes for free.
